1. Purpose & Scope

This manual explains how to use the Odoo Mail Client module in Odoo 18. It covers:

  • Day-to-day email tasks (read, compose, organize, send)
  • Setting up personal incoming and outgoing mail accounts
  • Administrator configuration and monitoring
  • Troubleshooting and support procedures

Intended for: Business users managing email communication, administrators, and support staff.

2. Module Overview

Module name: odoo_mail_client

Core capabilities:

  • Custom Email Management app inside Odoo
  • Mailbox features: Inbox, Sent, Drafts, Unread, Starred, Archived
  • Compose with To/Cc/Bcc and attachments
  • Personal incoming mail account management (IMAP)
  • Outgoing SMTP settings per account
  • Fetch tools: immediate refresh or date-range fetch (1/3/6 months)
  • Sync metrics and account health monitoring
  • Email signatures and tags
  • Privacy: Each user sees only their own accounts and emails

3. Quick Start (End User)

  1. Open the Email Management app from Odoo.
  2. If no incoming accounts exist yet, ask your administrator or create your own (see below).
  3. Use the mailbox list: Inbox, Sent, Draft, Starred, Unread, Archived.
  4. Open an email to read, reply, or forward.
  5. Use action buttons: Mark Read/Unread, Star, Archive, Delete.
  6. Compose new messages with To/Cc/Bcc and attachments.
Tip: Keep draft messages until you're ready to send. Use star and archive to keep your inbox organized.

4. End-User Guide

4.1 Mailbox Navigation

Available mailbox views:

  • All Emails: Every message across all accounts
  • Inbox: Incoming messages only
  • Sent: Outgoing messages you have sent
  • Draft: Unsent messages in progress
  • Unread: Messages not yet marked read
  • Starred: Messages you flagged for follow-up
  • Archived: Messages you have archived

Search and filter: Use keyword search for subject/body/sender/tag context. Use filters for unread, starred, with attachments. Group by priority or incoming account.

4.2 Reading Email

When opening a non-draft message, you can:

  • Mark Read / Mark Unread
  • Star / Unstar
  • Archive / Unarchive
  • Reply (to sender and thread)
  • Forward (to new recipients)
  • Delete (permanently remove)

Message metadata includes sender, recipients, date, thread references, priority, and tags.

4.3 Composing and Sending

  1. Create a new message in draft state.
  2. Fill To (required if Cc is empty).
  3. Optionally fill Cc and Bcc.
  4. Add Subject (defaults to "(No subject)" if left blank).
  5. Write body content (rich text editor).
  6. Attach files if needed.
  7. Select From Account if using multiple accounts.
  8. Click Send.

Send behavior: If you selected a From Account with SMTP settings, the message is sent through that account. Otherwise, Odoo may queue delivery depending on configuration.

4.4 Organizing Messages

  • Tags: Color-coded categories for messages (Finance, Support, Urgent, etc.)
  • Priority: Low, Medium, High, Very High
  • Star: Flag important messages for follow-up
  • Archive: Move completed conversations out of main inbox

5. Setting Up Your Own Mail Account

Each user can create and manage personal mail accounts. Follow this workflow to add an account:

5.1 Open Configuration

  1. In Email Management, go to Configuration → Incoming Mail Servers.
  2. Click Create to add a new account.

5.2 Fill Incoming Server Settings

In draft state, fill these fields:

  • Account Name: A friendly name (e.g., "Work Email", "Gmail")
  • Server: IMAP hostname (e.g., imap.gmail.com, mail.example.com)
  • Port: Usually 993 for IMAP + SSL, or 143 for plain IMAP
  • SSL/TLS: Check if your provider requires SSL connection
  • Username: Your email address or account login
  • Password: Your email password (typically required for IMAP)
  • Active: Check to enable this account
  • Delete On Server When Deleted In Odoo: Check if you want Odoo deletions to also delete emails from the server

5.3 Fill Outgoing SMTP Settings

In the same form, configure SMTP for sending:

  • SMTP Host: SMTP server hostname (e.g., smtp.gmail.com, mail.example.com)
  • SMTP Port: Typically 587 for STARTTLS, 465 for SSL, 25 for unencrypted
  • SMTP User: Your email address or account login
  • SMTP Password: Your email password for SMTP authentication
  • SMTP Encryption: Choose None, STARTTLS, or SSL/TLS based on provider
Example (Gmail):
IMAP: imap.gmail.com:993 (SSL)
SMTP: smtp.gmail.com:587 (STARTTLS)
Note: Gmail requires app-specific passwords if you have 2FA enabled.

5.4 Test and Confirm

  1. Click Test & Confirm.
  2. System validates IMAP connection and moves account to active (done) state.
  3. If test fails, check credentials, server name, port, and encryption settings.

5.5 Fetch Your First Emails

  1. In the confirmed account, click Fetch Now.
  2. A wizard opens; select fetch period: 1 month, 3 months, or 6 months back.
  3. Click Fetch.
  4. You receive a notification with count of checked and failed messages.

5.6 Reset and Reconfigure

If you need to change credentials later:

  1. Open the confirmed account.
  2. Click Reset Configuration.
  3. Update fields and click Test & Confirm again.

6. Administrator Features

6.1 Permissions and Security

User groups:

  • Internal Users: Can access email records based on ownership and record rules
  • Email Manager: Can create, update, delete mail accounts with ownership rules
  • System Administrator: Full access, but still scoped by privacy rules

Privacy Model: Each user sees only their own accounts and emails. Accounts are private by default; record rules enforce user ownership and login matching.

6.2 Signatures

Path: Email Management → Configuration → Signatures

  • Create reusable email signatures per account or user.
  • Define default signature for compose context.
  • Use HTML formatting for professional templates.

6.3 Email Tags

Path: Email Management → Configuration → Tags

  • Create custom tags for email categorization.
  • Assign colors for visual organization.
  • Users apply tags during message review.

6.4 Account Health Monitoring

Path: Email Management → Configuration → Account Health

Monitor sync metrics:

  • Last fetch start / end
  • Fetch duration (milliseconds)
  • Checked count and failed count
  • Last fetch error (if any)
  • Last synced UID and UID validity

Use this view daily: Check for failures, slow sync, and accounts never synced.

6.5 Custom Mail Logo (Optional)

In System Settings, admins can enable and upload a custom logo for the email client UI branding.

7. Troubleshooting

7.1 Test and Confirm Fails

Checks:

  • Verify IMAP hostname, port, SSL/TLS setting with provider.
  • Verify username and password are correct.
  • Confirm network/firewall allows IMAP port.
  • If using Gmail: use app-specific password, not your account password.

7.2 No Emails After Fetch

Checks:

  • Confirm account is active and in done state.
  • Run Fetch Now again and check notification for count.
  • Verify mailbox permissions and IMAP access on server.
  • Check Account Health for last fetch error details.

7.3 Cannot Send Email

Checks:

  • Confirm at least one valid recipient in To field.
  • Verify SMTP host, port, user, password, encryption with provider.
  • Confirm network/firewall allows SMTP port.
  • Check selected From Account has SMTP configured.

7.4 User Cannot See Expected Emails

Cause: Mailbox visibility is scoped by user ownership, incoming server ownership, or server login matching.

Resolution:

  • Verify you are logged in with the correct user account.
  • Verify the incoming server is created by or owned by you.
  • Verify server username matches your user login or email.
  • Contact your administrator if visibility is incorrect.

7.5 Access Errors

Common causes: Missing group assignment, restrictive record rules, insufficient ACL on models.

Resolution: Verify user group membership and contact administrator.

8. Operational Procedures

8.1 New Account Onboarding

  1. Create a new incoming server record.
  2. Enter incoming and outgoing settings in draft state.
  3. Click Test & Confirm.
  4. Use Fetch Now to retrieve initial emails (select 1, 3, or 6 months).
  5. Verify messages appear in Inbox.

8.2 Credential Rotation

  1. Open the account in done state.
  2. Click Reset Configuration to return to draft.
  3. Update IMAP and SMTP credentials and settings.
  4. Click Test & Confirm.
  5. Run Fetch Now and verify metrics update without errors.

8.3 Routine Health Checks (Daily Recommended)

  1. Review Account Health view for failures and slow sync.
  2. Check Last Fetch Error for any failing accounts.
  3. Verify recent message flow in Inbox and Sent.
  4. Contact users if their accounts are not syncing.

8.4 Backup and Recovery

  • Include email.record, fetchmail.server, mail.signature, and ir.attachment in database backups.
  • After restore, revalidate account credentials and connectivity.
  • Run account health review and sample send/fetch test.
